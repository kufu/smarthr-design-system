## ディレクトリ構成

`src` 以下のディクトリ構成について説明します。

- components
    - React コンポーネントを入れるディレクトリです
    - この中は export するコンポーネントの粒度でディレクトリを切ります
- hooks
    - React Hooks な関数を記述する場所です
- libs
    - 文字列操作みたいな汎用的な関数を記述する場所です
- themes
    - テーマの設計については後述します
- types
    - グローバルな型定義をする場所です


## 設計の基本方針

Atomic Design などのような難しい設計はしておらず、単純に SmartHR UI のユーザーが import するコンポーネントの単位がディレクトリの分割単位です(例外あり)。

例外として、 例えば Button というディレクトリからは PrimaryButton, SecondaryButton, TextButton といったようなテーマごとの Button コンポーネントを export していたりします。
コンポーネントの内部の記述として、基本的には難しい処理を持たせるべきではないですが、どうしても必要になる場合は hooks を使用することを推奨しています。
