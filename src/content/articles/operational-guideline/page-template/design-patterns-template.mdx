概要文として、主題の役割やドキュメントの内容の要約を見出しなしで書きます。
`description`より詳細な記述になることは問題ありませんが、同等の内容になるようにしてください。

## \{用語\}とは
用語の説明や定義が必要になる場合に書きます。
2-3行程度の短い説明で十分な場合は、セクションにせず上の概要文に含めることもできます。

## 基本的な考え方
主題を定義するうえでの基本方針や原則を書きます。概要的な説明はここには書きません。

冒頭に、このパターンが扱う範囲／扱わない範囲を1行で書きます。

```markdown
このパターンでは{扱う対象}を扱います。{扱わない対象}は{関連パターン名}で扱います。
```

関連パターン（特にアクセシビリティ方針）と矛盾・競合しうる点がある場合は、優先関係を表で書きます。

```markdown
| 関連パターン | 整合を要する点 | 優先 / 扱い |
| :--- | :--- | :--- |
| {関連パターン名} | {矛盾・競合しうる点} | {どちらを優先するか / このパターンでは扱わない旨} |
| {競合する代替コンポーネント} | {例: Heading と Fieldset legend のどちらを使うか} | {使う方・代替してよい条件} |
```

### アクセシビリティ
主題全体に対するアクセシビリティの考え方として、強調して言及する場合に書きます。  
要素に閉じている場合は、`{要素名}`のセクションに書きます。

### ライティング
主題全体に対する文言の考え方として、強調して言及する場合に書きます。  
要素に閉じていた内容は、`{要素名}`のセクションに書き、全体の文言のバリエーションは`ライティングパターン`に書きます。


## 構成
主題を構成する要素の全体像と、それぞれの要素を説明します。テンプレは以下です。

```markdown
{主題}は以下の要素で構成されています。

1. 要素名1
2. 要素名2
3. 要素名3

| 要素 | 役割 | 確定文言（UIに出る語） | コンポーネント / variant | 配置 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| {要素名1} | {役割} | `{確定文言}` | {コンポーネント} / {variant} | {順序・位置} | {可変部の決め方・参照先など} |
```

- UIに出る文言は「確定文言」列にそのまま使える形で書き、散文に埋めません。可変部は `{ }` で変数化し、決め方を備考に書きます。
- コンポーネント・variant・配置は、「通常ボタン／テキストボタン」「確定ボタンの左隣」など記憶・直感とぶつかりやすい指定ほど明示します。

*要素の全体を俯瞰する画像や`ComponentPreview`があればここに埋め込みます。*

### \{要素名\}
要素の役割や、要素のみに関する余白、種類、文言のルールや制約を、よしなに子セクションを分けて定義します。文言・コンポーネント・配置の確定指定は、冒頭の一覧表と整合させます。

実装を一意に決めるのに必要な具体値・挙動は、次の表で書きます。空欄や「資料に記述なし」を残さず、「規定しない（実装判断）」「未規定（WIP）」まで明記します。

```markdown
| 適用に必要な値・挙動 | 値 / 参照先 | 備考 |
| :--- | :--- | :--- |
| {状態→表現の対応} | {例: 在籍=`green` / 退職=`grey`} | {使用コンポーネントのprop名} |
| {状態遷移後の挙動} | {例: 中断で表示し、ステップ完了で消す} | {遷移後も残さない等} |
| {自動適用される値} | {例: 既定 `0.5`（8px）} | {実値を明記} |
| {基準値} | {例: `SCREEN_SMALL`（トークン参照）} | {数値でなく一意に辿れる参照で} |
```

グルーピングを扱う要素は、含める範囲と区別の表現を確定します。

```markdown
| グルーピング | 含める | 含めない | 区別の表現（具体値） |
| :--- | :--- | :--- | :--- |
| {まとまりの名前} | {例: コンテンツのみ} | {例: 見出し} | {背景色トークン等。地と同色になる場合の代替手段も併記} |
```

1つ要素の説明が長大になる場合は、それ自体を別のページにわけることを考えましょう。  
要素内に閉じない、全体に関わるものは`レイアウト`や`種類`などのセクションに書きます。

*要素の画像や`ComponentPreview`、スプレッドシートがあればここに埋め込みます。*


## レイアウト
主題を配置する際の余白のとり方や配置の制約を、よしなに子セクションを分けて説明します。  
`構成`で定義した要素間の配置関係もここに含みます。

余白・比率は、実値・方向（縦／横）・適用範囲・実現手段（コンポーネント／prop）まで書きます。

要素内に閉じたレイアウトは`{要素名}`のセクションに書きます。

*レイアウト全体を俯瞰する画像や`ComponentPreview`があればここに埋め込みます。*


## 種類
主題全体のUIバリエーションを説明します。

状態によって要素の出し分けがある場合は、冒頭に「状態 × 要素」マトリクスを必ず置きます。セルは「表示／非表示／差し替え」で書き、差し替えは差し替え後の内容へ参照します。

```markdown
| 状態 \ 要素 | {要素1} | {要素2} | {要素3} |
| :--- | :--- | :--- | :--- |
| 通常 | 表示 | 表示 | 表示 |
| {条件付きの出し分け（例: タイトルが見出しを兼ねる）} | 省略 | 表示 | 表示 |
| {レイアウト種別（例: フルスクリーン）} | 表示 | 表示 | 表示 |
| {別パターンへの変換（例: モバイルのリスト変換）} | 表示 | 差し替え → {参照} | 非表示 |
| {単体配置などの規則} | 非表示 | 表示 | 下部に表示 |
```

種類が選択肢のカタログ（ダイアログ種別など）の場合は、想定される条件を網羅し、カタログにない条件を残しません。

判断基準を示す場合は、本則・例外・優先関係を1つの表にまとめます。

```markdown
| 基準 | 本則 | 例外（適用条件） | 優先関係・備考 |
| :--- | :--- | :--- | :--- |
| {基準名} | {本則} | {例外と適用してよい条件} | {優先順位。例外の適用は理由を明示させる} |
```

### \{バリエーション名\}
どんなときのバリエーションか、その種類になる条件や役割を書きます。
バリエーションごとのライティングルールがあれば、`<blockquote>`やスプレッドシートなどを含めてここで書きます。

挙動・レイアウトに迷いが出る項目には、DoとDon'tの両例を汎用的な題材で書きます（Do例は必須）。

```markdown
#### 例
- ✅ Do: {汎用的な最小実装の説明・図}（なぜ良いか1行）
- 🚫 Don't: {ありがちな誤り}（なぜ不可か1行）
```

*バリエーションの画像や`<blockquote>`、`ComponentPreview`、スプレッドシートなどがあればここに埋め込みます。*


## ライティングパターン
主題に種類がなく、ライティングルールだけ書きたい場合はここに書きます。  
`<blockquote>`やスプレッドシートなども埋め込めます。抜粋元（ライティングガイドライン等）への一意に辿れる参照を書きます。

*ライティングガイドラインから抜き出した`<blockquote>`やスプレッドシートなどがあればここに埋め込みます。*


## モバイル
モバイルでの表示について、特に言及したい場合はここに書きます。

モバイル表示における構成やレイアウトなどを記載する場合は、基本的にこのセクションにまとめて書くことを推奨します。
ただし、各セクションのなかでモバイル表示について言及するほうが都合がよい場合は、それぞれのセクションの中に書くことも問題ありません。

切り替えの基準値はトークン（例: `SCREEN_SMALL`）で書き、`種類`のマトリクスにモバイルの行を置いた場合はこのセクションへの参照で繋ぎます。


## 受け入れチェックリスト
Yes/Noで答えられる粒度で書きます。数値上限・配置・出し分け・文言一致・禁止ルール（Don't）・例外適用の明示・単位や文言の一貫性を含めます。

```markdown
- [ ] {数値上限（例: 操作エリアのボタンは最大3つ）}
- [ ] {出し分け（例: 初期0件で「並べ替え」を非表示にしている）}
- [ ] {確定文言との一致（例: トリガーのラベルが「一括操作」になっている）}
- [ ] {禁止ルール（例: タブを「絞り込みなど表示状態の切り替え」に使っていない）}
- [ ] {例外適用の明示（例: 本則と異なる場合、理由が実装コメントに書かれている）}
- [ ] {一貫性（例: 件数の単位がオブジェクトに合っており、実装内で一貫している）}
```

## \{その他の基準\}
基本的に上記の項目に当てはめることを推奨しますが、主題がルールのみの場合など、これらに当てはまらない内容を説明する場合に使用する自由項目です。  
よしなに子セクションを分けて書きます。
