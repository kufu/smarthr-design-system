Making rich and interactive features accessible can be extremely challenging. 
This article provides examples of best practices for common rich interactive features.

## 1. Drag and Drop Lists

### Provide keyboard accessibile single pointer controls for drag and drop actions

Drag and drop lists are a common feature in web applications, but attention needs to be taken to ensure that they are accessible.
One requirement for accessible drag and drop actions is that they also need to be able to be operated via keyboard, according to WCAG 2.5.7 (For any action that involves dragging, provide a simple pointer alternative.)

A common way this is achieved is by providing buttons to move items up and down the list, or through options in an extended ellipsis menu.

Also see WCAG 2.1.1 (Keyboard) for more information on keyboard accessibility.

### Draggable Regions

When implementing drag and drop features, it is important to ensure that the draggable region is properly identified and does not lead to nested interactive controls. 

### Pointer Cancellation

When implementing drag and drop features, ideally the drag and drop action itself should be cancellable by the user. A common implementation of this is to cancel the drag and drop action by using the Escape key on the keyboard.
If an action is not cancellable, then an alternative method is to provide a method to undo the latest action, such as an undo button or option in a menu. 
This is handled by WCAG 2.5.2 (Pointer Cancellation).

#### Common pitfalls to avoid
- Do not nest interactive controls. For example, if your draggable region has the role of button, do not nest html button elements within it. This can cause confusion for assistive technologies and breaks WCAG 4.1.2 (Name, Role, Value).
Instead, use ARIA attributes to properly identify the draggable region as "draggable" whilst still keeping it as a div element, and use ARIA live regions to provide feedback on the drag and drop action. Or, change the draggable area to use a drag handle that is separate from the interactive card component.
