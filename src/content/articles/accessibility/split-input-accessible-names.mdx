## 分割インプットとは
分割インプットとは、可視ラベル1つに対して複数の入力フォームが紐づいているインプットフォームを指します。

分割インプットは、主に以下の2つに分類されます。

* 1つの項目を複数の入力フィールドに分割して入力する場合
* レンジなどを入力するために、複数の入力フィールドを用意する場合

### アクセシブルネームとは

アクセシブルネーム(Accessible Name)に関しては、 [アクセシブルネーム(Accessible Name)とは](./check-list/accessible-name/#h2-1) を参照してください。

### 分割インプットのアクセシビリティ的な問題点
分割インプットでは、複数の入力フィールドに対して1つのラベルしかないため、以下の達成基準を満たせていません。

* [フォームパーツにアクセシブルネームがある](./check-list/accessible-name) 
* [入力する内容や、操作がラベルとして表示されている](./check-list/label)

そのため、できる限り分割インプットの仕様を避け、それぞれのインプットフォームにラベルをつけるようにしてください。

SmartHRでは、紙の様式に則ってデータ分割をしていたり、画面上の情報量の関係で、分割インプットフォームを使用している箇所があります。
そのため、分割インプットを使用する場合のアクセシブルネームの付け方について、以下のルールを定めています。


## 分割インプットのアクセシブルネームの書き方

### 1つの項目を複数の入力フィールドに分割して入力する場合

1つの項目を複数の入力フィールドに分割して入力する場合、アクセシブルネームは「ラベル　分割位置の説明」という形式で指定します。

- 2つに分割する場合: 「ラベル最初のN桁」「ラベル残りのN桁」
- 3つに分割する場合: 「ラベル最初のN桁」「ラベル次のN桁」「ラベル最後のN桁」

例えば、「郵便番号」を2つの入力フィールドに分ける場合


```jsx
<Fieldset title="郵便番号">
  <Cluster>
    <Input aria-label="郵便番号 最初の3桁" />
    <span>-</span>
    <Input aria-label="郵便番号 残りの4桁" />
  </Cluster>
</Fieldset>
```

#### 電話番号などの特殊なケース

電話番号のように場所によって桁数が変わる場合は、桁数を指定せず、代わりに「N/全体数」という形式を使用します:

- 例: 「電話番号 1/3」「電話番号 2/3」「電話番号 3/3」

{/* ![電話番号の分割入力例](./images/placeholder-phone-split-input.png) */}

```jsx
<Fieldset title="電話番号">
  <Cluster>
    <Input aria-label="電話番号 1/3" />
    <span>-</span>
    <Input aria-label="電話番号 2/3" />
    <span>-</span>
    <Input aria-label="電話番号 3/3" />
  </Cluster>
</Fieldset>
```
