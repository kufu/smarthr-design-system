`fieldset`要素の代替として複数の入力要素をグルーピングするコンポーネントです。関連する入力欄を1つの見出しでまとめるときに使います。

姓名や期間の開始・終了のように値を分割して入力する場合や、[Checkbox](/products/components/check-box/)や[RadioButton](/products/components/radio-button/)といった複数の入力要素を持つフォームに対してラベルやメッセージテキストを表示したい場合に適しています。

<ComponentStory name="Fieldset" />

## 使用上の注意

### 入力要素が2つ以上の場合にのみ使用する

Fieldsetは、2つ以上の入力要素を持つフォームをグループ化し、タイトルやラベル、メッセージテキストを表示したい場合に使用します。

また、タイトルでは、まとめた入力要素がどんなグループであるかを明確に示しましょう。

入力要素が1つだけの場合は、[FormControl](/products/components/form-control/)を使用してください。

## レイアウト

Fieldsetには、複数の入力要素のグループの目的を明確にするタイトルをつけます。

Fieldsetに内包されるフォーム要素のタイトルは、グルーピングしているFieldsetのタイトルよりも低い見出しレベルになるようにします。見出しに関する詳細は[Heading](/products/components/heading/)を参照してください。

また、FormControlと同様に、ステータスラベルやメッセージテキストの有無で、コンポーネント内のレイアウトにバリエーションがあります。詳細は[FormControlのレイアウト](/products/components/form-control/#h2-1)を参照してください。

### エラーの表示方法

Fieldsetに含まれる入力要素にエラーがある場合、エラーメッセージをそれぞれのFormControlではなくFieldsetのタイトルの下に表示できます。これによって、入力要素の組み合わせによって発生する複合的なエラーを表示したり、FormControlを横に並べたときのレイアウト崩れを防いだりできます。  

このとき、エラーメッセージには該当のFormControlのタイトルを含めてください。

### 基本的なアクセシビリティ機能

#### グループの意味的な構造化
Fieldsetコンポーネントは、複数の入力要素を`<fieldset>`要素でグループ化し、`legend`属性で指定したテキストを`<legend>`要素として出力します。これにより、支援技術を使用するユーザーにも、フォーム要素のグループとその目的が伝わります。

#### スクリーンリーダーでの読み上げ
グループ内の各入力要素にフォーカスすると、スクリーンリーダーは「[グループ名]、[ラベル名]」のように読み上げます。これにより、ユーザーは現在入力している項目がどのグループに属しているかを理解できます。

### 開発時の考慮点

#### 複数の入力要素をグループ化する際は必ずFieldsetを使用する
CheckboxやRadioButtonなど、複数の関連する入力要素をグループ化する際は、必ずFieldsetコンポーネントを使用してください。単に視覚的にグループ化するだけでなく、支援技術にもグループの構造が伝わるようにすることが重要です。

#### legendでグループの目的を明確に示す
`legend`属性には、グループ化された入力要素の目的を明確に示すテキストを設定してください。例えば、「通知設定」「住所」「性別」のように、グループ全体が何を表しているのかがわかるテキストにします。

#### CheckboxやRadioButtonのグループには必ずFieldsetを使用する
CheckboxやRadioButtonの選択肢をグループとして提示する場合は、必ずFieldsetでグループ化してください。グループ化しないと、スクリーンリーダーを使用するユーザーは、複数の選択肢がどのような質問や設定に対するものなのか理解できません。

#### 良い実装例

```tsx
// CheckboxまたはRadioButtonのグループ
<Fieldset
  legend="通知設定"
  innerMargin={0.5}
>
  <Cluster gap={{ row: 1, column: 1.5 }}>
    <Checkbox name="emailNotification">メール通知</Checkbox>
    <Checkbox name="smsNotification">SMS通知</Checkbox>
    <Checkbox name="pushNotification">プッシュ通知</Checkbox>
  </Cluster>
</Fieldset>

// 複数のFormControlをグループ化
<Fieldset
  legend="住所"
  helpMessage="ご自宅の住所を入力してください"
>
  <Stack gap={1}>
    <FormControl label="郵便番号">
      <Input name="postalCode" />
    </FormControl>
    <FormControl label="都道府県">
      <Select
        name="prefecture"
        options={[
          { label: '都道府県を選択してください', value: '' },
          // ...
        ]}
      />
    </FormControl>
  </Stack>
</Fieldset>
```

#### 悪い実装例

```tsx
// ❌ Fieldsetを使用せずdivでグループ化
<div>
  <Text styleType="blockTitle">通知設定</Text>
  <Cluster gap={{ row: 1, column: 1.5 }}>
    <Checkbox name="emailNotification">メール通知</Checkbox>
    <Checkbox name="smsNotification">SMS通知</Checkbox>
  </Cluster>
</div>

// ❌ legendが曖昧
<Fieldset legend="設定">
  <Checkbox name="emailNotification">メール通知</Checkbox>
  <Checkbox name="smsNotification">SMS通知</Checkbox>
</Fieldset>
```

Fieldsetを使用せずdivでグループ化すると、支援技術にグループの構造が伝わりません。また、legendが曖昧だと、ユーザーは何の設定なのか理解しにくくなります。

### 読み取り専用

Fieldset自体には、読み取り専用の状態は存在しません。

入力内容の確認時など、すでに入力済みの書き換えできない値として表示する際は、[Input](/products/components/input/)の`readOnly`を使用してください。

ただし、値をテキストとしてコピーして利用したいユーザーにとっては、分割されているフォームの値をそれぞれコピーする必要があり、手間になってしまうことに注意してください。

```tsx editable
<Fieldset
    legend={{ text: "電話番号", styleType: "subBlockTitle" }}
    innerMargin={0.5}
>
    <Cluster gap={0.5}>
        <FormControl label={{ text: "電話番号1", unrecommendedHide: true }}>
            <Input
                width="100px"
                name="phoneNumber1"
                value="00"
                readOnly
            />
        </FormControl>
        <FormControl label={{ text: "電話番号2", unrecommendedHide: true }}>
            <Input
                width="100px"
                name="phoneNumber2"
                value="0000"
                readOnly
            />
        </FormControl>
        <FormControl label={{ text: "電話番号3", unrecommendedHide: true }}>
            <Input
                width="100px"
                name="phoneNumber3"
                value="0000"
                readOnly
            />
        </FormControl>
    </Cluster>
</Fieldset>
```
