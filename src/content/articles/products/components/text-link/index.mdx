`a`要素の代わりに使用する汎用テキストリンクコンポーネントです。本文中にリンクを置くとき、[HelpLink](/products/components/text-link/help-link/)や[UpwardLink](/products/components/text-link/upward-link/)で表現できないテキストリンクを提供するときに使います。

## 使用上の注意

### ボタンとして使わない

`TextLink`は、ページを移動するためのリンクです。それ以外の処理を実行するボタンとして使わないでください。

視覚的優先度の低いボタンを表現したい場合は、[Tertiaryボタン](/products/components/button/#h3-3)を使ってください。

## レイアウト

### アイコンの有無

特定の役割を示すためにアイコンを使います。これはテキストが異なっていても、同じ挙動であることを示すためです。

#### アイコン付きテキストリンク

テキストリンクの左右にアイコンを配置できます。
SmartHR UIでは、`アイコン付き（左）`はプレフィックス（`prefix` props）に、`アイコン付き（右）`はサフィックス（`suffix` props）にIconを指定することで表現できます。  
アイコンは左右どちらかにのみ指定してください。

##### アイコン付き（左）

テキストリンクを押した移動先の意味・内容を想起させるために使用します。

##### アイコン付き（右）

テキストリンクを押したときの挙動を表現するために使用します。

### 基本的なアクセシビリティ機能

#### アクセシブルネームの提供
TextLinkコンポーネントは、children（子要素）に渡したテキストが自動的にアクセシブルネームとして設定されます。これにより、スクリーンリーダーを使用するユーザーにもリンクの目的が伝わります。

#### 新規ウィンドウで開く場合の自動通知
`target="_blank"`を指定した場合、TextLinkは自動的に新規ウィンドウで開くことを示すアイコン（OpenInNewTabIcon）を表示します。これにより、すべてのユーザーに別タブで開くことが視覚的に伝わります。

#### キーボード操作
- <kbd>Tab</kbd>キーでフォーカスの移動が可能
- <kbd>Enter</kbd>キーでリンク先への移動が可能
- フォーカス状態が視覚的に明確に表示される

### 開発時の考慮点

#### リンクテキストは文脈なしで理解できるようにする
リンクテキストは、周囲の文脈がなくても、それ単体でリンク先や目的が理解できるようにしてください。スクリーンリーダーのユーザーは、ページ内のすべてのリンクを一覧表示して目的のリンクを探すことがあります。

#### 情報的アイコンには適切な説明を提供する
suffix（サフィックス）にダウンロードアイコンなど情報を伝えるアイコンを配置する場合、必要に応じて`aria-label`でアイコンの意味を含めた説明を提供してください。ただし、新規ウィンドウで開くアイコンは自動的に処理されるため、個別の設定は不要です。

#### 可視テキストとアクセシブルネームを一致させる
`aria-label`を使用して追加情報を提供する場合でも、必ず可視テキストを含めてください。音声入力を使用するユーザーは、画面に表示されているテキストで操作を指示するため、可視テキストが含まれていないと操作できません。

#### 良い実装例

```tsx
// 基本的なテキストリンク
<TextLink href="/employees">従業員一覧</TextLink>

// 新規ウィンドウで開く（自動的にアイコンが表示される）
<TextLink href="https://example.com" target="_blank">
  外部サイト
</TextLink>

// 文脈を含む具体的なリンクテキスト
<TextLink href="/employees/12345">山田太郎さんの詳細</TextLink>
```

#### 悪い実装例

```tsx
// ❌ 曖昧なリンクテキスト（何の詳細か不明）
<TextLink href="/employees/12345">詳細</TextLink>

// ❌ 生のa要素を使用（新規ウィンドウアイコンが自動表示されない）
<a href="https://example.com" target="_blank">
  外部サイト
</a>

// ❌ 可視テキストを含まないaria-label
<TextLink href="/search" aria-label="サイト内を検索">
  検索
</TextLink>
// 正しくは: aria-label="検索: サイト内を検索"

// ❌ 「クリック」などの操作を含む表現
<TextLink href="/help">クリックしてヘルプを表示</TextLink>
```

曖昧なリンクテキストは、リンク一覧から目的のリンクを探す際に判別できません。また、生のa要素を使用すると、新規ウィンドウで開くことが視覚的に伝わらず、ユーザーが予期しない動作に驚く可能性があります。

## デザインパターン

### 特別なテキストリンク

特定の役割を示すためにアイコンを使います。これはテキストが異なっていても、同じ挙動であることを示すためです。

#### 新規ウィンドウで開くテキストリンク 

テキストリンクで別の画面に移動することで、現在の画面での入力や作業が中断され作業効率が著しく落ちる場合には、新規ウィンドウ（新規タブ）でテキストリンクを開くことを検討します。

- TextLinkは、**テキストの右側**に <OpenInNewTabIconInTextLink /> アイコン（`OpenInNewTabIcon`）を自動で配置するため、個別に設定する必要はありません。
    - リンク先がSmartHRの内部ページ、外部サイトに関わらず表示します。
- `アイコン付き（左）`（プレフィックス）は指定しないでください。

```tsx editable
  <TextLink href="/" target="_blank">新規ウィンドウで開くテキストリンク</TextLink>
```

#### ヘルプセンターを開くテキストリンク

[ヘルプセンター](https://support.smarthr.jp/ja/)を開くテキストリンクは[HelpLink](/products/components/text-link/help-link/)として提供しています。

#### 一階層上のコンテンツに戻るテキストリンク

一階層上のコンテンツに戻るテキストリンクはスタイリングやpropsを整えたコンポーネント[UpwardLink](/products/components/text-link/upward-link/)として提供しています。`アイコン付き（右）`（サフィックス）は指定しないでください。

```tsx editable
  <UpwardLink href="/">オブジェクト一覧に戻る</UpwardLink>
```
