単一の入力要素にラベル、ヘルプ/エラー/補足のメッセージテキスト、入力必須か否かを紐づけるためのコンポーネントです。1つの入力欄にラベル付けするとき、入力欄にエラーや補足メッセージを表示するときに使います。

<ComponentStory name="FormControl" />

## 使用上の注意

### 入力要素が1つの場合にのみ使用する

FormControlは、入力要素が1つだけの場合にのみ使用できます。

姓名のように値を分割して入力する場合や、CheckboxやRadioButtonといった複数の入力要素を持つフォームに対してラベルやメッセージテキストを表示したい場合は、[Fieldset](/products/components/fieldset/)を使用してください。

#### 複数のフォームを横に並べる場合

入力要素をグルーピングできる場合は、基本的に[Fieldset](/products/components/fieldset//)を使用してください。エラーが表示された際のレイアウト崩れを防ぐことができます。（[エラーの表示方法](/products/components/fieldset#h3-1)）

表示領域の都合などでやむを得ずFormControlを横に並べる場合は、エラーの表示時にレイアウトが崩れないか注意してください。

## レイアウト

ステータスラベルやメッセージテキストの有無で、コンポーネント内のレイアウトにバリエーションがあります。

### ステータスラベルの有無

フォーム入力が入力必須か任意かをユーザーが区別できるようにするために使用します。
代表的なステータスラベルは次のとおりです。

| 種類 | 表示例 | 説明 |
| :--- | :--- | :--- |
| `必須` | <StatusLabel type="red">必須</StatusLabel> | 入力必須のフォームの場合に表示します。 |
| `任意` | <StatusLabel type="grey">任意</StatusLabel> | 任意のフォームの場合に表示します。 |

#### フォーム入力が必須の場合の表示例

```tsx editable
<FormControl
  label="タイトル"
  statusLabels={<RequiredLabel />}
>
  <Input name="defaultInput" />
</FormControl>
```

#### フォーム入力が任意の場合の表示例

```tsx editable
<FormControl
  label="タイトル"
  statusLabels={<StatusLabel type="grey">任意</StatusLabel>}
>
  <Input name="defaultInput" />
</FormControl>
```

#### ステータスラベルの省略

ステータスラベルは視覚的に強い注意を惹くことから、認知負荷を減らすために省略する場合があります。
具体的には、以下のケースには省略を検討してください。

##### 必須を省略するケース

- フォーム入力が極めて少なく、すべて必須であることが自明な場合
- 任意のフォームよりも入力必須のフォームが多い場合

##### ステータスラベル自体を省略するケース

- フォームの説明文などで、すべての項目が必須、または任意であることが明記されている場合

### メッセージテキストの有無

以下の種類別のメッセージテキストを目的に応じて組み合わせて使用できます。

| 種類 | Props | 説明 |
| :--- | :--- | :--- |
| ヘルプメッセージ | `helpMessage` | 入力して欲しい内容を具体的に伝えるテキストを入力要素の上に表示します。必要に応じて関連するヘルプページなどへのテキストリンクを設置できます。テキストリンクの詳細は[ヘルプページへの動線](/products/contents/ui-text/help-link/#h3-2)を参照してください。 |
| 入力例 | `exampleMessage` | フォームの入力例を入力要素の上に表示します。原則として入力要素の`placeholder`はユーザビリティおよびアクセシビリティの観点から非推奨としているため、その代替として使用してください。 |
| エラーメッセージ | `errorMessages` | フォームバリデーションなどで、適切な値が入力されていないことを伝えるテキストを、<FaCircleExclamationIcon alt="エラーメッセージ" color="DANGER" /> アイコン（`FaCircleExclamation`）とあわせて入力要素の上に表示します。 |
| 補足メッセージ | `supplementaryMessage` | 入力時にユーザーが把握しなくても操作に影響のない情報を入力要素の下に表示します。 |

#### メッセージテキストの表示例

```tsx editable
<FormControl
  label="タイトル"
  helpMessage="ヘルプメッセージ"
  exampleMessage="入力例"
  errorMessages="エラーメッセージ"
  supplementaryMessage="補足メッセージ"
>
  <Input name="input" error />
</FormControl>
```

#### エラーメッセージ

エラーメッセージは、入力必須の入力要素の値が空の場合や、データ型が異なるなどの適切な値が入力されていない場合に表示します。

どの入力要素がエラーとなっているかを視覚的に見つけやすくするために、入力要素にも`error`状態を付加します。

同一画面上でエラーをリアルタイムに判定できる場合は、エラーの原因が解消されたらメッセージの表示を止めます。

##### 初期状態の場合

```tsx editable
<FormControl
  label="氏名"
  helpMessage="氏名を入力してください。"
  exampleMessage="入力例：須磨栄子"
  supplementaryMessage="その他ご不明な点がありましたら、サポートまでお問い合わせください。"
>
  <Input name="fullname" width="100%" />
</FormControl>
```

##### 適切な値が入力されていない場合

```tsx editable
<FormControl
  label="氏名"
  helpMessage="氏名を入力してください。"
  exampleMessage="入力例：須磨栄子"
  errorMessages="氏名が入力されていません。"
  supplementaryMessage="その他ご不明な点がありましたら、サポートまでお問い合わせください。"
>
  <Input error name="fullname" width="100%" />
</FormControl>
```

### 基本的なアクセシビリティ機能

#### ラベルと入力要素の自動関連付け
FormControlコンポーネントは、`label`属性で指定したテキストを`<label>`要素として出力し、内包する入力要素（Input、Select、Textareaなど）と自動的に関連付けます。これにより、以下のメリットがあります：

- スクリーンリーダーを使用するユーザーに入力の目的が伝わる
- ラベルをクリック/タップすることで入力要素にフォーカスできる（操作領域が広がる）
- 音声入力でラベルテキストを使って操作できる

#### エラーメッセージの関連付け
`errorMessages`属性を設定すると、エラーメッセージが入力要素と適切に関連付けられます。スクリーンリーダーでフォーカス時に、エラー状態とエラーの内容が自動的に読み上げられます。

#### ヘルプメッセージ・補足メッセージの関連付け
`helpMessage`や`supplementaryMessage`も入力要素と関連付けられ、スクリーンリーダーで読み上げられます。

### 開発時の考慮点

#### 入力要素には必ずFormControlを使用する
Input、Select、Textareaなどの入力要素を使用する際は、必ずFormControlで囲んでラベルを提供してください。ラベルがないと、スクリーンリーダーを使用するユーザーは何を入力すべきか理解できません。

- eslint-plugin-smarthrの[smarthr/a11y-input-in-form-control](https://github.com/kufu/eslint-plugin-smarthr/tree/main/rules/a11y-input-in-form-control/)ルールは、InputにFormControlを組み合わせることを促すものです。有効にして使用することを推奨します。

#### 可視ラベルを必ず提供する
`aria-label`のみでラベルを提供するのではなく、必ず可視ラベル（画面に表示されるラベル）を使用してください。可視ラベルには以下のメリットがあります：

- すべてのユーザーが視覚的に入力の目的を理解できる
- 音声入力を使用するユーザーが画面に表示されているテキストで操作できる
- 認知負荷が低い

#### placeholderをラベル代わりに使用しない
`placeholder`属性は入力例を示すための補助的な機能であり、ラベルの代わりにはなりません。以下の理由から、必ず`label`属性で可視ラベルを提供してください：

- placeholderは入力中に消えてしまう
- スクリーンリーダーで安定して読み上げられない環境がある
- 視認性が低い（コントラストが低い）
- 入力済みの値とplaceholderが区別しにくい

入力例を示したい場合は、`exampleMessage`属性を使用してください。

#### 良い実装例

```tsx
// 基本的な使い方
<FormControl label="メールアドレス">
  <Input name="email" type="email" />
</FormControl>

// ヘルプメッセージと入力例を含む
<FormControl
  label="氏名"
  helpMessage="姓と名の間にスペースを入れてください"
  exampleMessage="入力例：山田 太郎"
>
  <Input name="fullName" type="text" />
</FormControl>

// エラー状態
<FormControl
  label="メールアドレス"
  errorMessages="正しいメールアドレスの形式で入力してください"
>
  <Input name="email" type="email" error />
</FormControl>
```

#### 悪い実装例

```tsx
// ❌ FormControlを使用せずラベルを別要素で表示
<Stack gap={0.5}>
  <Text styleType="blockTitle">メールアドレス</Text>
  <Input name="email" type="email" />
</Stack>

// ❌ placeholderのみでラベルを提供
<Input
  name="email"
  type="email"
  placeholder="メールアドレス"
/>

// ❌ aria-labelのみでラベルを提供
<Input
  name="email"
  type="email"
  aria-label="メールアドレス"
/>

// ❌ エラーメッセージが入力要素と関連付けられていない
<Stack gap={0.5}>
  <FormControl label="メールアドレス">
    <Input name="email" type="email" />
  </FormControl>
  <div style={{ color: 'red' }}>
    正しいメールアドレスの形式で入力してください
  </div>
</Stack>
```

FormControlを使用しない場合、ラベルと入力要素が関連付けられず、スクリーンリーダーを使用するユーザーには入力の目的が伝わりません。また、エラーメッセージが適切に関連付けられていない場合、フォーカス時にエラー状態が通知されません。

## 状態

### 無効（disabled）

フォームの操作ができない状態を表現する際は、[Input](/products/components/input/)など内包する要素の`disabled`を使用してください。

ユーザーはなぜ無効になっているのかわからないことがあります。[権限による表示制御](/products/design-patterns/access-control-pattern/)のデザインパターンを参考にして、そもそも無効ではなくフォーム自体を非表示にしたり、無効状態の理由を付近に表示することを検討してください。

```tsx editable
<FormControl
  label="氏名"
  helpMessage="氏名を入力してください。"
  exampleMessage="入力例：須磨栄子"
  supplementaryMessage="その他ご不明な点がありましたら、サポートまでお問い合わせください。"
>
  <Input name="fullname" width="100%" disabled />
</FormControl>
```

### 読み取り専用


入力内容の確認時など、すでに入力済みの書き換えできない値として表示する際は、[Input](/products/components/input/)など内包する要素の`readOnly`を使用してください。

```tsx editable
<FormControl label="氏名">
  <Input name="fullname" value="草野栄一郎" readOnly />
</FormControl>
```
