余白トークンは、主にレイアウトで使われるトークンです。

CSSでは`margin`や`padding`, `top`, `left`, `bottom`, `right`が関連します。

## 基準値
ベースの基準サイズは`16px`です。

## プリミティブトークン

| トークン名 | 値（rem） | 値（px） |
| :--- | :--- | :--- |
| `0`    | 0       | 0    |
| `0.25` | 0.25rem | 4px  |
| `0.5`  | 0.5rem  | 8px  |
| `0.75` | 0.75rem | 12px |
| `1`    | 1rem    | 16px |
| `1.25` | 1.25rem | 20px |
| `1.5`  | 1.5rem  | 24px |
| `2`    | 2rem    | 32px |
| `2.5`  | 2.5rem  | 40px |
| `3`    | 3rem    | 48px |
| `4`    | 4rem    | 64px |

### `0.25`

**説明**

`StatusLabel`やアイコンなどの要素をテキストと組み合わせる場合
- `InformationPanel`や`FormControl`、`FieldSet`、`TextLink`を利用することで、自動的に適用されます。その他の場合は、`Cluster`を使用して余白を指定してください。

#### 他のトークンとの使い分け

- **`0.5`（8px）**: StatusLabelやアイコンなどの要素をテキストと組み合わせる場合

#### 注意事項

- StatusLabelやアイコンなどの要素をテキストと組み合わせる場合、明確な理由がない場合、原則として`0.5`（8px）を利用します。

### 0.5

#### 使いどころ

- StatusLabelやアイコンなどの要素をテキストと組み合わせる場合

#### 他のトークンとの使い分け

- **`0.25`（4px）**: StatusLabelやアイコンなどの要素をテキストと組み合わせる場合
- **`1`（16px）**:
  - ブロックの見出しやInputにラベルや説明テキストを複合的に組み合わせる場合
  - Button同士の間

**注意事項**

- StatusLabelやアイコンなどの要素をテキストと組み合わせる場合、明確な理由がない限り、原則として**0.5（8px）**を利用します。
- Button同士の間のmarginを指定する際、原則として**1（16px）**を利用します。

### 0.75

**注意事項**

**[モバイル]** 0.5（8px）では余白が小さく、1（16px）では余白が大きい場合に0.75（12px）を利用する。

### 1

**説明**

- セクション内の見出しとコンテンツの間
- フォーム要素内の説明テキストとフォームの間
- 各セクションの見出しとコンテンツの間
- `UpwardLink`と画面タイトルの間
- ブロックの見出しや`Input`にラベルや説明テキストを複合的に組み合わせる場合
  - `FormControl`や`Fieldset`を利用することで、自動的に適用されます。
- `Checkbox`または`RadioButton`の`row-gap`
- フォームを横並びにする際の`row-gap`
- `Button`同士の間
  - 0.5（8px）のトークンもあります。
- 代表的なページ内レイアウトのパディング
  - [モバイル] メインコンテンツのパディング：`Container`を利用することで、自動的に適用されます。`padding` propsで値を指定できます。`UpwardLink`がある場合は、`UpwardLink`がある場合のパディングを参照して、`Container`の`padding` propsを指定してください。
  - [モバイル] コンテンツエリアのパディング：`Base`の`padding` propsで値を指定してください。
  - コンテンツエリア内のグループのパディング：`BaseColumn`を利用することで、自動的に適用されます。
- 代表的なダイアログ内レイアウトのパディング
  - [モバイル] コンテンツエリアのパディング：`Dialog`の各種コンポーネントを利用することで、自動的に適用されます。値は`contentPadding` propsで指定できます。
  - コンテンツエリア内のグループのパディング：`BaseColumn`を利用することで、自動的に適用されます。
- 共通`Table`においてオブジェクトの数が0のとき、メッセージ下端と`Button`の距離
- 共通`List`においてオブジェクトの数が0のとき、メッセージ下端と`Button`の距離
- `Table`の`TitleArea`とテーブルの上端の間のマージン

**他のトークンとの使い分け**

- **0.5（8px）**
  - Button同士の間
- **1.5（24px）**
  - **[モバイル]** 代表的なページ内レイアウトのメインコンテンツのパディング

**注意事項**

- Button同士の間のmarginを指定する際、原則として**1（16px）**を利用します。

### 1.5

**説明**

- 代表的なページ内レイアウトのマージン
  - (必要に応じて) セクション同士の間
    - 視覚的に余白が大きく見える場合のマージン
  - セクション内の要素同士の間
- 代表的なダイアログ内レイアウトのマージン
  - フォーム要素同士の間
- タイトルエリアに `InformationPanel` を配置しない場合のタイトルエリアと最初のセクションの間
- `Checkbox` や `RadioButton` の `column-gap`
  - `Cluster` を使用して余白を指定してください
- [モバイル]メインコンテンツのパディング
  - `Container` を使用することで、自動的に適用されます。`padding` propsで値を指定できます。`UpwardLink` がある場合は、`UpwardLink` がある場合のパディングを参照して `Container` の `padding` propsを指定してください。
- コンテンツエリアのパディング
  - `Base` の `padding` propsに値を指定してください。
- 代表的なダイアログ内レイアウトのパディング
  - コンテンツエリアのパディング
    - `Dialog` の各コンポーネントを使用することで、自動的に適用されます。`contentPadding` propsで値を指定できます。
- [モバイル] `UpwardLink` がある場合の `padding-top`
  - 代表的なページ内レイアウトのパディング
- `Table` の下部と `Pagination` のマージン

**他のトークンとの使い分け**

- **1（16px）**
  - メインコンテンツのパディング
- **2（32px）**
  - セクション同士の間

**注意事項**

セクション同士の `margin` を指定する際、原則として**2（32px）**を利用

### 2

**説明**

- **セクション同士の間**
  - 必要に応じて1.5（24px）（視覚的に余白が大きく見える場合のマージンを参照）
- **メインコンテンツのパディング**
  - `Container`を使用することで、自動的に適用されます。`padding` propsで値を指定できます。`UpwardLink`がある場合は、`UpwardLink`がある場合のパディングを参照して`Container`の`padding` propsを指定してください。
- **代表的なダイアログ内レイアウトのマージン**
  - 説明テキストとフォーム要素のセクションの間
- **`UpwardLink`がある場合の`padding-top`**
  - 代表的なページ内レイアウトのパディング

**他のトークンとの使い分け**

**1.5（24px）**: セクション同士の間

**注意事項**

セクション同士のmarginを指定する際、原則として2（32px）を利用

### 3

**説明**

- よくあるリストのオブジェクトの数が0件の場合、`Base`の`Top`からテキストまでの距離
- よくあるリストのオブジェクトの数が0件の場合、`Base`の`Bottom`から`Button`までの距離
- よくあるリストのオブジェクトの検索結果や絞り込み結果が0件であった場合、`Base`の`Top`と`Bottom`からテキストまでの距離

### 4

**説明**

- よくあるテーブルのオブジェクトの数が0件の場合、`Base`の`Top`からテキストまでの距離
- よくあるテーブルのオブジェクトの数が0件の場合、`Base`の`Bottom`から`Button`までの距離
- よくあるテーブルのオブジェクトの検索結果や絞り込み結果が0件であった場合がない場合、`Base`の`Top`と`Bottom`からテキストまでの距離
