影のトークンは、要素の周りに落とす影（シャドウ）の大きさを定義します。

CSSでは`box-shadow`や`drop-shadow`を指しています。

## 値の考え方

影の大きさや柔らかさ、拡散量によって要素間の高低差を表現できます。

例えば値の小さい影を持つ要素は、その背後にある要素と近いことを表しています。より大きく柔らかい影は、要素間の距離が離れていることを表します。

それぞれの要素に高低差の視覚効果を与えることで、インターフェース全体に以下のメリットが生まれます。

* ユーザーが要素間の優先度を読み取りやすく、どの要素がインタラクティブなのかを理解しやすくします。
* 要素のレイヤー順序を明確に定義することで、要素同士の意図しない干渉や重なりを回避します。

## セマンティックトークン

`LAYER0`は影なし。
色は[透明色](/products/design-tokens/color/#h3-6)を使用しています。

| トークン名 | 値（`box-shadow`） |
| :--- | :--- |
| `LAYER0` | none |
| `LAYER1` | 0 1px 2px 0 [`TRANSPARENCY_30`](/products/design-tokens/color/#h3-6) |
| `LAYER2` | 0 2px 4px 1px [`TRANSPARENCY_30`](/products/design-tokens/color/#h3-6) |
| `LAYER3` | 0 4px 8px 2px [`TRANSPARENCY_30`](/products/design-tokens/color/#h3-6) |
| `LAYER4` | 0 8px 16px 4px [`TRANSPARENCY_30`](/products/design-tokens/color/#h3-6) |

### `LAYER0`

例外的に、コンポーネントの影を打ち消したい場合に使います。Baseの影を打ち消す場合は、`layer`propsに`0`を指定します。

#### 使用上の注意

- `BACKGROUND`の上に置く`Base`には使わない
  - `LAYER0`は影なしのため、`BACKGROUND`との境界が視覚的に区別できなくなります。`layer`を省略するか`LAYER1`以上を使ってください。

### `LAYER1`

背景の上に表示する視覚的グルーピングのための矩形に使います。

コンポーネント内部での使用例：[AppNavi](/products/components/app-navi/)、[Base](/products/components/base/#h3-0)

### `LAYER2`

以下の場合に使います。

- マウスオーバー時のみ表示される要素
    - コンポーネント内部での使用例：[Tooltip](/products/components/tooltip/)
- 矩形（主にBase）を垂直方向または水平方向に分割し、それぞれが異なるスクロール領域を持つ場合

### `LAYER3`

以下の場合に使います。

- スクロールに関わらず画面内に固定表示する要素
    - コンポーネント内部での使用例：[FloatArea](/products/components/float-area/)
- 一時的に要素の上に重なって表示され、ユーザーによる明示的な操作で閉じられる要素
    - コンポーネント内部での使用例：[Calendar](/products/components/calendar/)、[Dropdown](/products/components/dropdown/)
- 一時的に既存の要素間に割り込んで表示される要素を視覚的に目立たせたい場合
    - コンポーネント内部での使用例：[InformationPanel](/products/components/information-panel/)

### `LAYER4`

例外的に、`LAYER3`を用いても背景とのコントラストが十分でない場合に使います。
